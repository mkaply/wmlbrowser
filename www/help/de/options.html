Einstellungen Wmlbrowser

In diesem Abschnitt:
  • Web-Servern die Darstellung von WML-Inhalten anbieten
  • Laden lokaler WML-Dateien

Folgende Einstellungen können in der Wmlbrowser-Version 0.7 vorgenommen werden:

Web-Servern die Darstellung von WML-Inhalten anbieten

Was bedeutet das?

Bei jedem Laden einer Web-Seite sendet Ihr Browser einen sogenannten "Accept"-Parameter mit, der die vom Browser darstellbaren Inhaltsformate beschreibt.

Wenn Sie diese Einstellung aktivieren, wird Ihr Browser auch WML-Seiten zur Darstellung akzeptieren.

Dies kann bedeuten, daß der Web-Server Ihnen andere Inhalte ausliefert!

Welche Einstellung ist empfehlenswert?

Der empfohlene Standard-Wert ist "aus". Wenn Sie die Einstellung aktivieren, kann dies dazu führen, daß Sie andere als die erwarteten Seiten erhalten. (Hotmail hat hier bekanntermaßen Probleme.)

Firefox-Nutzer können auch eine "WML-Modus"-Schaltfläche zur Werkzeugleiste hinzufügen, um diese Einstellung zu ändern. (Rechtsklick auf die Werkzeugleiste, "Anpassen..." und dann die "WML-Modus"-Schaltfläche (kleines Handy-Bildchen) auf die Werkzeugleiste ziehen.)

SeaMonkey-Nutzer können die Schaltfläche im "Wmlbrowser"-Ast des Einstellungsdialogs aktivieren.

Sie können auch eine Werkzeugleisten-Schaltfläche für diese Einstellung verwenden.

Laden lokaler WML-Dateien

Wo liegt das Problem?

Zwar erlaubt die Wmlbrowser-Erweiterung Mozilla/Firefox die Darstellung der WML-Seiten eines Web-Servers, aber die Dateiendung ".wml" ist dem Browser in der Regel unbekannt.

Wenn Sie diese Einstellung aktivieren, weiß Ihr Browser, daß ".wml"-Dateien als WML-Inhalte darzustellen sind.

Welche Einstellung ist empfehlenswert?

Lassen Sie diese Einstellung "aus", solange Sie keine ".wml" aus Ihrem lokalen Dateisystem laden wollen.
