options de wmlbrowser

Dans cette section:
  • Faire savoir aux sites que ce navigateur comprend le format WML
  • Chargement des fichiers depuis le système de fichiers.

Voici les options de wmlbrowser version 0.7.

Faire savoir aux sites que ce navigateur comprend le format WML

Ce que cela signifie :

Lorsque vous envoyez une requête pour une page sur le web, votre navigateur envoie ce que l'on appelle un en-tête d'acceptation, qui décrit les types de données que le navigateur sait gérer.

Si vous activez cette option, votre navigateur fera savoir aux sites web qu'il comprend le WML.

Cela peut entraîner l'affichage d'une page différente que celle que vous attendiez.

Ce qui est recommandé:

La meilleure option par défaut est la désactivation. L'activation de cette option peut faire en sorte que vous receviez des pages que vous n'attendiez pas, et il a été rapporté qu'elle empêcherait l'accès à Hotmail.

Les utilisateurs de Firefox peuvent ajouter le bouton « Mode WML » sur la barre d'outils pour modifier cette option plus aisément. (Cliquez avec le bouton droit de votre souris sur la barre d'outils, choisissez « Personnaliser », puis glissez et déposez le bouton WML (icône représentant un téléphone mobile) sur cette barre).

Les utilisateurs de SeaMonkey peuvent activer le bouton dans la section wmlbrowser de la fenêtre Préférences.

Chargement des fichiers depuis le système de fichiers.

Quel est le problème ?

Par défaut, l'extension wmlbrowser permet à Mozilla/SeaMonkey/Firefox de charger des pages WML depuis un serveur web et de les afficher. Mais si vous avez un fichier doté d'une extension ".wml" sur l'un de vos disques locaux (disque dur, disquette, CD, etc...) , votre navigateur ne saura pas comment le gérer.

Si vous activez cette option, votre navigateur saura qu'un fichier local .wml contient des données WML et l'affichera correctement.

Quel est le choix par défaut recommandé ?

Je conseille de laisser cette option désactivée sauf si vous savez que vous devrez charger des fichiers WML depuis votre système de fichiers.
